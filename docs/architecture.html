2 Technology Stack

Layer Technology Why this choice Alternative considered
UI Framework React 18 + Vite Component model fits role-based UI; Vite gives instant HMR Next.js (overkill — no SSR needed)
Styling Tailwind CSS v4 Utility-first, no build step for CSS, design tokens baked in CSS Modules / Styled Components
Charts Recharts React-native, responsive, zero config for bar/line/pie charts Chart.js (imperative API, less React-idiomatic)
Backend Node.js 24 + Express Minimal overhead, same language as frontend, huge ecosystem FastAPI / Django (separate language, heavier)
Authentication HMAC-signed tokens Stateless — no Redis/session store needed; tamper-proof JWT (larger token size); Sessions (requires store)
Database SQLite (node:sqlite) Zero-cost, single file, zero config, built into Node 24 — no extra dependency PostgreSQL (separate server, cost); MongoDB (no relations)
Routing React Router v6 Declarative nested routes with role guards; well-known API TanStack Router (newer, less docs)

3 Performance Cycle — Data Flow

1

Goal Creation

Employee adds up to 8 goals with thrust area, target, weightage (must sum to 100%)

2

Submission

Sheet submitted to manager. Status changes: draft → submitted

3

Approval

Manager reviews, can adjust targets, then approves & locks or returns for rework

4

Quarterly Tracking

Q1–Q4: Employee logs actual values. System auto-scores progress 0–150%

5

Reports & Audit

CSV export, analytics dashboards, full audit trail, escalation engine

6 Role-Based Access Control

👨‍💼 Employee
  • Create & edit own goals (while draft)
  • Submit goal sheet to manager
  • Log quarterly achievements
  • View own progress scores
  • View shared goals pushed by manager
🏢 Manager
  • Review all team members' sheets
  • Adjust targets before approving
  • Approve & lock or return for rework
  • Write quarterly check-in feedback
  • Push shared goals to team members
  • View team analytics
🔑 Admin (HR)
  • Configure FY cycle & quarter dates
  • View all sheets across org
  • Unlock any approved/locked sheet
  • Download CSV achievement reports
  • View full audit trail
  • Run escalation engine

7 Scoring Logic

📐 Per-Goal Score (0–150%)

Computed automatically when an achievement is logged:


numeric_min / percent actual ÷ target (higher = better)
numeric_max target ÷ actual (lower = better)
timeline on-time → 100%, late → 50%
zero-based actual = 0 → 100%, else 0%

⚖️ Sheet Score (weighted average)

Each goal's score is multiplied by its weightage. Sum across all goals gives the overall sheet score.

sheetScore = Σ ( goalScorei × weightagei ) / 100

Unachieved goals contribute 0%. Score caps at 150% per goal to reward stretch targets.

8 Hosting Strategy

💡

Zero-Infrastructure Design

The entire application is designed to run on a single small VM, container, or free-tier hosting service — no managed database, no message queue, no session store.

1
Node.js process (API)
1
Static build (Frontend)
1
SQLite file (DB)
$0
DB hosting cost

🐋 Docker / Container

Both processes in a single container image. The atomquest.db file is mounted as a volume for persistence. Ideal for any cloud container service (Fly.io, Railway, Render, etc.).

Recommended

☁️ Render.com

A render.yaml is already included in the repo. One web service for the API, one static site for the frontend build. Free tier is sufficient for a team of ~100.

Config included

🖥️ Single VM

Run both processes with PM2 or systemd on any Linux VM (e.g., AWS t2.micro free tier). Nginx reverse proxy serves the frontend build and proxies /api to port 4000.

Classic

💻 Local Development

Two terminals: cd server && npm start and cd client && npm run dev. The Vite proxy forwards /api requests to :4000 automatically.

Dev mode

9 Escalation Engine

Rule-based flags are computed when an Admin triggers a check. Each rule has a threshold and an escalation chain.

Rule Threshold Escalation Chain
Goals not submitted 7 days after cycle goal window opens L1 → Employee
Manager hasn't approved 7 days after employee submission L2 → Manager
Check-in overdue Quarter window open but no check-in recorded L2 → Manager
Repeated non-compliance Escalation unresolved at L2 L3 → HR / Admin
